# ロックファイルの削除方法

## 問題

`.next/dev/lock`ファイルが存在するため、新しい開発サーバーが起動できない。

## 解決方法

### 方法1: ロックファイルを削除（推奨）

#### Step 1: .nextフォルダを削除

1. **VS Codeのエクスプローラーを開く**
2. **`.next`フォルダを探す**
   - 見つからない場合は、「表示」→「隠しファイルを表示」をチェック
3. **`.next`フォルダを右クリック → 「削除」**

#### Step 2: 開発サーバーを再起動

```cmd
npm run dev
```

---

### 方法2: プロセスを強制終了（Windows）

プロセス23972を強制終了する場合：

1. **タスクマネージャーを開く**
   - **Ctrl + Shift + Esc** を押す
   - または、**Ctrl + Alt + Delete** → 「タスクマネージャー」

2. **プロセスを探す**
   - 「詳細」タブを開く
   - 「PID」列を表示（表示されていない場合は、列見出しを右クリック → 「PID」を選択）
   - PID 23972を探す
   - または、「node.exe」や「next」という名前のプロセスを探す

3. **プロセスを終了**
   - プロセスを選択
   - 「タスクの終了」ボタンをクリック

#### Step 2: 開発サーバーを再起動

```cmd
npm run dev
```

---

### 方法3: コマンドラインでプロセスを終了（上級者向け）

1. **管理者権限でコマンドプロンプトを開く**
   - Windowsの検索で「cmd」と入力
   - 「コマンドプロンプト」を右クリック → 「管理者として実行」

2. **プロセスを終了**
   ```cmd
   taskkill /PID 23972 /F
   ```

3. **開発サーバーを再起動**
   ```cmd
   cd C:\Users\taka-\OneDrive\デスクトップ\project\survey-app
   npm run dev
   ```

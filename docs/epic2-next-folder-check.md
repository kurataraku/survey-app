# Epic2: .nextフォルダの確認方法

## .nextフォルダが見当たらない場合

### 隠しフォルダとして表示されていない可能性

`.next`フォルダは通常、隠しフォルダとして表示されます。以下の方法で確認できます：

#### 方法1: PowerShellで確認

```powershell
Get-ChildItem -Path . -Filter ".next" -Force
```

#### 方法2: エクスプローラーで確認

1. エクスプローラーでプロジェクトフォルダを開く
2. 「表示」タブをクリック
3. 「隠しファイル」にチェックを入れる
4. `.next`フォルダが表示されるか確認

### .nextフォルダが存在しない場合

`.next`フォルダが存在しない場合、削除されたか、まだ作成されていません。

この状態で開発サーバーを起動すると、新しい`.next`フォルダが自動的に作成されます。

### ロックエラーが続く場合

`.next`フォルダが存在しないのにロックエラーが出る場合：

1. **別のNodeプロセスが実行中**
   - タスクマネージャーでNodeプロセスを確認
   - すべてのNodeプロセスを終了

2. **ファイルシステムのキャッシュ**
   - 数秒待ってから再度試す
   - PCを再起動（最後の手段）

3. **開発サーバーを起動してみる**
   - `.next`フォルダが存在しない状態で`npm run dev`を実行
   - 新しいフォルダが作成され、正常に起動する可能性がある

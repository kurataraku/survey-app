# 開発サーバーの再起動手順

## Cursorのターミナルで実行する場合

Cursorのターミナルで開発サーバーを再起動することは**全く問題ありません**。以下の手順に従ってください。

## 手順

### Step 1: 現在実行中の開発サーバーを停止

1. **Cursorのターミナルウィンドウを見る**
   - 画面下部にターミナルが表示されているはずです
   - または、「表示」→「ターミナル」でターミナルを表示

2. **開発サーバーが実行中か確認**
   - ターミナルに以下のような表示が出ていれば、開発サーバーが実行中です：
     ```
     ▲ Next.js 16.1.1 (Turbopack)
     - Local:         http://localhost:3000
     - Network:       http://192.168.x.x:3000
     ✓ Ready in Xms
     ```

3. **開発サーバーを停止**
   - ターミナルウィンドウをクリックしてフォーカスを当てる
   - **`Ctrl + C`キーを押す**（Windowsの場合）
   - ターミナルに「^C」と表示され、開発サーバーが停止します
   - プロンプト（`>` または `$`）が表示されれば停止成功です

   **注意**: 複数回`Ctrl + C`を押す必要がある場合があります

### Step 2: プロジェクトディレクトリに移動

1. **現在のディレクトリを確認**
   - ターミナルに表示されているパスを確認
   - 以下のようなパスになっているはずです：
     ```
     C:\Users\taka-\OneDrive\デスクトップ\project\survey-app>
     ```

2. **正しいディレクトリにいない場合**
   - 以下のコマンドを実行して、プロジェクトディレクトリに移動：
     ```cmd
     cd C:\Users\taka-\OneDrive\デスクトップ\project\survey-app
     ```
   - または、VS Codeのエクスプローラーで`survey-app`フォルダを右クリック → 「ターミナルで開く」

### Step 3: 開発サーバーを起動

1. **以下のコマンドを実行**
   ```cmd
   npm run dev
   ```

2. **起動を確認**
   - ターミナルに以下のような表示が出れば、起動成功です：
     ```
     ▲ Next.js 16.1.1 (Turbopack)
     - Local:         http://localhost:3000
     - Network:       http://192.168.x.x:3000
     ✓ Starting...
     ✓ Ready in Xms
     ```

3. **エラーが出た場合**
   - エラーメッセージの内容を確認
   - よくあるエラー：
     - `Port 3000 is in use`: 別のプロセスがポート3000を使用している
     - `package.json not found`: 正しいディレクトリにいない

### Step 4: ブラウザで確認

1. **ブラウザを開く**
   - `http://localhost:3000/schools` にアクセス

2. **ページを完全にリロード**
   - `Ctrl + F5` を押す（Windowsの場合）
   - または `Ctrl + Shift + R`
   - これで、ブラウザのキャッシュを無視して最新のコードが読み込まれます

## よくある問題

### 問題1: `Ctrl + C`を押しても停止しない

**解決方法**:
1. ターミナルウィンドウをクリックしてフォーカスを当てる
2. 再度`Ctrl + C`を押す
3. それでも停止しない場合は、ターミナルを閉じて新しいターミナルを開く

### 問題2: ポートが使用中というエラー

**解決方法**:
1. 別のターミナルウィンドウで開発サーバーが実行されている可能性があります
2. すべてのターミナルウィンドウを確認
3. または、別のポートを使用する：
   ```cmd
   npm run dev -- -p 3001
   ```

### 問題3: 正しいディレクトリにいない

**解決方法**:
1. ターミナルで`cd`コマンドを使って、プロジェクトディレクトリに移動：
   ```cmd
   cd C:\Users\taka-\OneDrive\デスクトップ\project\survey-app
   ```
2. または、VS Codeのエクスプローラーで`survey-app`フォルダを右クリック → 「ターミナルで開く」

### 問題4: `npm run dev`が実行できない

**解決方法**:
1. Node.jsがインストールされているか確認：
   ```cmd
   node --version
   ```
2. `package.json`が存在するか確認：
   ```cmd
   dir package.json
   ```
3. 依存関係がインストールされているか確認：
   ```cmd
   npm install
   ```

## 確認チェックリスト

- [ ] ターミナルで`Ctrl + C`を押して開発サーバーを停止した
- [ ] プロジェクトディレクトリ（`survey-app`）にいることを確認した
- [ ] `npm run dev`を実行した
- [ ] ターミナルに「✓ Ready」というメッセージが表示された
- [ ] ブラウザで`http://localhost:3000/schools`にアクセスした
- [ ] ブラウザで`Ctrl + F5`を押してページを完全にリロードした

## 次のステップ

開発サーバーが正常に起動したら、学校カードをクリックして、コンソールログとサーバーログを確認してください。

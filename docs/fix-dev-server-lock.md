# 開発サーバーのロック問題の解決方法

## 問題

以下のエラーメッセージが表示される:
```
⨯ Unable to acquire lock at C:\Users\taka-\OneDrive\デスクトップ\project\survey-app\.next\dev\lock, is another instance of next dev running?
```

これは、既に別の開発サーバーが実行中であることを示しています。

## 解決方法

### Step 1: 既存の開発サーバーを停止

1. **他のターミナルウィンドウを確認**
   - VS Codeで開いている他のターミナルウィンドウを探す
   - `npm run dev`を実行しているウィンドウを探す

2. **そのターミナルで Ctrl + C を押して停止**
   - 全ての開発サーバーのインスタンスを停止してください

3. **プロセスが停止するまで待つ**
   - ターミナルにプロンプト（`C:\Users\...>`）が表示されるまで待つ

### Step 2: ロックファイルを削除（オプション）

もし、既存のプロセスを停止してもまだエラーが出る場合は：

1. **VS Codeのエクスプローラーで`.next`フォルダを探す**
   - 隠しファイルを表示する必要がある場合は、「表示」→「隠しファイルを表示」

2. **`.next/dev/lock`ファイルを削除**
   - または、`.next`フォルダ全体を削除

### Step 3: 開発サーバーを再起動

ターミナルで以下を実行:

```cmd
npm run dev
```

### Step 4: 正しいポートを確認

ターミナルに以下のようなメッセージが表示されます:

```
▲ Next.js 16.1.1
- Local:        http://localhost:3000
- Ready in X.Xs
```

**重要**: ポートが3002になっている場合は、ブラウザで以下にアクセスしてください:
```
http://localhost:3002
```
